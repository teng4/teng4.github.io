# How To display 2 robots in the same RViz environment?

## Steps-1

(1) make sure two publishers are in separate namespace: `"/omni1/joint_states"`, `"/omni2/joint_states"`. **Importantly**, when constructing the msg, the joint names should be exactly the same as the ones used in the corresponding URDF files, robot2 joint names could be `"omni2/waist"`, `"omni2/shoulder"`, `"omni2/elbow"`, etc.

## Steps-2

(2) In the URDF file of robot2, add namespace for all links and joints, e.g., `"omni2/waist"`, `"omni2/shoulder"`, `"omni2/elbow"`, etc. Note that, for robot1, you can add and use namespace as well like what we did for robot2, but for simplicity, the robot1 URDF and its joint state publisher can remain unchanged which means it is not necessary to add namespace for robot1 URDF and its joint state publisher.

## Steps-3

(3) Note, it is better to NOT use `<remap from="/tf" to="omni2/tf" />` and `<remap from="/tf_static" to="omni2/tf_static" />` in the launch file. Because it will make the command `"$ rosrun rqt_tf_tree rqt_tf_tree"` be invalid when you want to check the tf tree. `/tf` itself can discriminate the joint states based on namespace, so no need to remap `/tf`.

## Steps-4

(4) To move the base location of robot2 in RViz environment, use `<node pkg="tf2_ros" type="static_transform_publisher" name="robot2_base_transform" args="0.1 0 0 0 0 0 world omni2/base" />` in the lauch file, which will move the robot2 base to `x=0.1m`. Note that, (a) the `"omni2/base"` is according to the name you defined in the URDF file. (b) this static base transform requires to set the `"Fixed Frame"` in rviz file to be `"world"` instead of `"base"` (`"base"` in the base name used in URDF file).

## Steps-6

(6) Adjust the pose of the two robots in rviz using a mouse, and save the rviz file (This will auto correct some minor mistakes you may have made when editing the rviz file, e.g., the robot2 links should be `"omni2/base:"`, `"omni2/lower_arm:"`, etc.).


## Steps-7

(7) Finally, in the launch file, add the following code to launch the two robots with identified by different namespaces.

```
<launch>
  <!-- load the controllers -->
  <!-- <arg name="reference_frame" default="/map"/>   
  <arg name="units" default="mm"/>               
  <arg name="publish_rate" default="1000"/>

  <node name="omni_state6" pkg="omni_common" type="omni_state6" output="screen"> 
    <!-- <param name="omni_name" value="$(arg omni_name)"/>

    <param name="reference_frame" value="$(arg reference_frame)"/>
    <param name="units" value="$(arg units)"/> -->
  </node>

  <!-- Launch Robot 1 -->
  <group ns="ooomni11">

    <node pkg="robot_state_publisher" type="robot_state_publisher" name="robot_state_publisher" />
    <!-- <node name="robot_state_publisher" pkg="robot_state_publisher" type="robot_state_publisher" ns="ooomni" output="screen" /> -->

  </group>
```
